# 📊 통계처리입문 팀프로젝트

본 프로젝트는 사용자의 직관적 경험을 데이터와 통계적 근거로 검증하고, 이를 IT 서비스의 UX 개선안으로 연결하는 프로세스를 담고 있습니다. 

## 👤 Identity & Goals
* **Major**: IT+Management
* **Objective**: 실생활 데이터를 통계적으로 검증하여 사용자 경험(UX)을 향상시킬 수 있는 데이터 기반 의사결정 역량을 확보합니다.

## 🛠 Tech Stack
* **Language**: `R`
* **Core Skills**: 설문 설계, 데이터 전처리, 가설 검정(t-test), 서비스 기획(UX)

## 📂 Project Structure & Analysis

### 1. 가설 수립 및 데이터 수집
* **연구 주제**: 기상 조건(강수, 결빙, 기온)이 공용자전거 이용 의향에 미치는 영향 분석
* **데이터 수집**: 공용자전거 이용 경험자 20명을 대상으로 설문조사 수행 (2025.12.7. ~ 2025.12.9.)

### 2. 통계적 가설 검정 (t-test)
* **[가설 1] 강수 강도에 따른 차이**: 보통 비 상황보다 이슬비일 때 이용 의향이 통계적으로 유의미하게 높음을 증명했습니다.
* **[가설 2] 결빙 위험 인식**: 눈이 내리는 중일 때보다 눈이 온 후 도로가 얼어붙었을 때 이용 의향이 훨씬 낮게 나타났습니다.
* **[가설 3] 이용 목적별 저항력**: 예상과 달리 통학/출근 그룹과 운동 그룹 간의 악천후 시 이용 의향 차이는 통계적으로 유의미하지 않았습니다.
* **[가설 4] 온도 임계점**: 사용자는 추위보다 더위에 더 민감하게 반응하여 이용을 포기하는 경향이 있음을 확인했습니다.

### 3. 실무적 서비스 개선안 (UX Insight)
* **세분화된 날씨 알림**: 단순 강수 여부가 아닌 '이슬비' 등 강수 강도에 따른 이용 가능성 안내 서비스.
* **안심패스 서비스**: 노면 온도 센서 데이터를 활용하여 실시간 결빙 위험 구간을 지도에 시각화하여 제공.
* **폭염 대응 인프라**: 분석 결과를 바탕으로 폭염 시기 전기자전거 확대 및 그늘막 대여소 우선 배치 전략 제안.

## 💡 Key Experience
* **통계적 사고의 확장**: 지난 통계청 공모전에서의 거시적 산업 분석 경험을 바탕으로, 이번에는 실제 원천 데이터를 수집하고 $p$-value 해석을 통해 미시적인 서비스 개선 우선순위를 도출했습니다.
* **데이터 기반 서비스 기획**: 분석 결과를 수치에 그치지 않고, 사용자 페인 포인트(Pain Point)를 해결하는 구체적인 IT 기능으로 구체화했습니다.

## 📋 Survey Design
| 항목 | 내용 및 측정 척도 |
| :--- | :--- |
| **공용자전거 이용 여부** | 예 / 아니오 선택 |
| **주된 이용 목적** | 통학(통근) / 운동 및 산책 / 기타 중 택 1 |
| **일주일 평균 이용 횟수** | 주관식 입력 |
| **기상 조건별 이용 의향** | 0~5점 리커트 척도 (강수량별, 강설/결빙별) |
| **이용 포기 임계 온도** | 주관식 입력 (더운 경우 / 추운 경우) |

## 🔒 Data Transparency & Ethics
* **Privacy Compliance**: 수집 당시 명시된 데이터 활용 범위(학업 목적)를 준수하기 위해 원본 설문 데이터는 비공개 처리하였습니다.
* **Reproducibility**: 코드의 실행 흐름을 확인할 수 있도록 가공된 샘플 데이터(`sample_data.csv`)를 함께 첨부하였습니다.

### ⚠️ Note on Sample Data & Results
* **Original Analysis**: 본 프로젝트의 실제 결론은 설문 응답자 20명의 전체 데이터를 바탕으로 도출되었습니다.
* **Sample Data**: 현재 레포지토리에 업로드된 `sample_data.csv`는 데이터 보안 및 개인정보 보호를 위해 임의로 추출된 5개의 샘플만을 포함하고 있습니다.
* **Result Discrepancy**: 샘플 데이터의 크기가 작아 가설 검정 결과($p$-value)가 실제 분석 결과와 다를 수 있으나, 이는 코드의 정상 작동 여부를 확인하기 위한 용도임을 밝힙니다.
